# Testimonial & Social Proof Blocks

Templates for embedding social proof in proposals. Use real testimonials from the user when available — use placeholders when not.

---

## Block Formats

### Format 1: Quote + Attribution
```
"{{Quote — 1-2 sentences, focused on a specific result or experience}}"

— {{Name}}, {{Title/Role}}, {{Company}}
```

Best for: Most proposals. Clean, credible, professional.

### Format 2: Mini Case Study
```
CLIENT: {{Name/Company}}
CHALLENGE: {{What they were dealing with — 1 sentence}}
SOLUTION: {{What you did — 1 sentence}}
RESULT: {{Specific outcome — metric, timeframe, or transformation}}
```

Best for: Data-driven clients, technical proposals, corporate style.

### Format 3: Before/After
```
BEFORE: {{Their situation before working with you}}
AFTER: {{Their situation after — specific improvement}}
TIMEFRAME: {{How long it took}}
```

Best for: Transformation-focused services, coaching, consulting.

### Format 4: Stat Block
```
{{METRIC}} — {{context}}
Example: "34% increase in qualified leads within 60 days"
Example: "Reduced onboarding time from 3 weeks to 4 days"
Example: "Generated $47K in new revenue from the campaign"
```

Best for: Minimal style proposals, B2B, performance marketing.

---

## Placement in Proposals

| Location | Purpose | Format |
|----------|---------|--------|
| After Executive Summary | Reinforce credibility early | Quote or Stat Block |
| After Scope of Work | Show you've done this before | Mini Case Study |
| Before Pricing | Justify the investment | Before/After or Stat Block |
| After Pricing | Reassure after seeing the number | Quote |
| In About Section | Build trust in your credentials | Quote from a notable client |

**Rule:** 1-2 testimonials per proposal is ideal. More than 3 feels desperate.

## Placeholder Templates

When the user doesn't have testimonials yet:

### Placeholder A: Named Slot
```
"{{PLACEHOLDER: Insert a testimonial from a past client in a similar
industry. Focus on results — specific metrics or transformations.}}"

— {{Client Name}}, {{Title}}, {{Company}}
```

### Placeholder B: Generic Social Proof
```
We've helped {{number}} {{clients/businesses/professionals}} achieve
{{type of result}}. {{Brief elaboration or example.}}
```

### Placeholder C: Credential-Based
```
With {{X}} years of experience in {{field}} and projects spanning
{{industries/types}}, we bring {{specific_expertise}} to every engagement.
```

> **AGENT: When using placeholders:**
> 1. Use Placeholder B or C (credential-based) as defaults — they don't require specific testimonials
> 2. Flag every placeholder: "⚠️ Placeholder testimonial — replace with a real quote before sending"
> 3. If the user says "I don't have testimonials," use Placeholder C and suggest they collect reviews from past clients

---

## Writing Tips for Testimonial Selection

When the user provides multiple testimonials, pick the one that:

1. **Matches the project type.** A web design testimonial for a web design proposal, not a generic "great to work with."
2. **Has specific results.** "$47K in revenue" beats "really happy with the work."
3. **Comes from a similar client.** A testimonial from a startup CEO for a startup client proposal.
4. **Is recent.** A 2025 testimonial beats a 2020 one.
5. **Is concise.** 1-2 sentences is ideal. Edit long testimonials down (with permission).
