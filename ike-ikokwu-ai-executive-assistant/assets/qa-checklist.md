# QA Checklist

Run every proposal through this checklist before delivery.

---

## Content Accuracy

- [ ] Client name spelled correctly everywhere
- [ ] Company name spelled correctly
- [ ] Project title is clear and descriptive
- [ ] Executive summary references the actual conversation/meeting
- [ ] Scope items match what was discussed (nothing invented)
- [ ] All deliverables are listed and described
- [ ] Timeline is realistic and includes buffer
- [ ] Pricing math is correct (tiers, totals, payment splits)
- [ ] Payment terms add up to 100%
- [ ] No real competitor or brand names used

## Completeness

**All proposal types:**
- [ ] Cover page present (client, provider, date, validity)
- [ ] Executive summary written (not a placeholder)
- [ ] About/credentials section
- [ ] Social proof (testimonials or placeholders)
- [ ] Terms & conditions (appropriate to type and size)
- [ ] Next steps with clear CTA

**Service proposals additionally:**
- [ ] Scope of work with specific deliverables
- [ ] Out-of-scope section included
- [ ] Timeline with milestones
- [ ] Pricing with tiers (or single price if user specified)

**Partnership proposals additionally:**
- [ ] Both parties' roles explicitly defined
- [ ] Mutual benefits listed (not one-sided)
- [ ] Partnership duration and terms
- [ ] Revenue share or contribution model (if applicable)

**Membership proposals additionally:**
- [ ] Membership benefits clearly described
- [ ] Tiers differentiated by access/value (if multiple)
- [ ] Billing cycle and cancellation terms
- [ ] Enrollment process described

**Sponsorship proposals additionally:**
- [ ] Audience data with specific numbers
- [ ] Sponsorship levels with clear deliverables per tier
- [ ] Materials deadline and fulfillment terms
- [ ] Category exclusivity stated

**Speaking proposals additionally:**
- [ ] Topic options with descriptions
- [ ] Format options (keynote, workshop, etc.)
- [ ] Speaker fee + expenses separated
- [ ] Logistics requirements listed
- [ ] Recording/distribution rights addressed

## Formatting

- [ ] Consistent style throughout (font, spacing, headers)
- [ ] Matches selected proposal style (corporate, consultant, etc.)
- [ ] Short paragraphs, scannable sections
- [ ] No walls of text
- [ ] Section headers are clear and descriptive
- [ ] Tables used for timeline and pricing (clean, aligned)

## Persuasion

- [ ] More "you/your" than "we/our" (client-focused language)
- [ ] Benefits emphasized over features in scope descriptions
- [ ] Executive summary tells a story: problem → approach → outcome
- [ ] Pricing comes AFTER scope and deliverables (value first)
- [ ] Good/better/best options are properly differentiated
- [ ] Recommended option is clearly marked
- [ ] Guarantee or risk reversal included (if applicable)

## Placeholders

- [ ] All {{placeholder}} variables are clearly marked with ⚠️
- [ ] Placeholder testimonials are flagged for replacement
- [ ] User knows what they need to fill in before sending

## Final Checks

- [ ] Read-through: reads smoothly from cover to next steps
- [ ] The "would I hire this person?" test — does the proposal inspire confidence?
- [ ] Proposal validity date is set (typically 30 days)
- [ ] Contact info is included in next steps
- [ ] HTML page renders correctly (if built)
- [ ] Score is 80+ on the rubric
